Octopus数据接口OpenApi使用说明

3.3 供“渠道商”调用服务操作接口

3.3.1 渠道商获取访问token

  • 服务地址

    http://<ip>:<port>/channel/service/gettoken

  • 请求方式

    POST

  • 输入参数

    序号 参数名 类型 释义 是否必须
    1 susername String 渠道商用户名 是
    2 spassword String 渠道商密码 是

  • 返回示例

    成功会返回包含token（获取到的token）和expires_in（有效期）的json数据包，具体示例如下：

    {"token":"TOKEN","expires_in":7200}

  • 失败则返回

    {
    "code": "404" //200是作为服务成功返回代码,失败则为404
    "msg": "错误消息"
}

3.3.2 注册新的服务(可填写”服务商”id)

  • 服务地址

    http://<ip>:<port>/channel/service/reg?channeltoken=CHANNELTOKEN

  • 请求方式

    POST

  • 输入参数

    序号 参数名 类型 释义 是否必须
    0 sid String 服务Id，1.不填则默认注册新服务,2.填则会先查找旧服务，查到就做更新，查不到就做插入（新sid做插入） 否
    1 sname String 服务名 是
    2 surl String 服务地址 否
    3 simage String 图片地址 否
    4 des String 服务描述 是
    5 token String 远端服务验证用token：注册服务时由用户提供，和唯一服务关联（作为服务注册数据的一部分），无生命周期，平台访问远端时，会作为访问参数的一部分(经过加密后融合进sigature) 否
    6 providerid String 服务商id,1.填,则会将当前服务指定该服务商下 否

  • 输出

    { 
    "sid": "服务id",
    "appId": "这里是appid，后续需要作为申请access_token的参数",
    "secrityId": "这里是secrityId，后续需要作为申请access_token的参数",
    "channelid": "渠道商id",
    “providerid”:”服务商id”
    "code": "200" //200是作为服务成功返回代码,失败则为404
    "msg": "错误消息"
}

3.3.3 删除服务

  • 服务地址

    http://<ip>:<port>/channel/service/del?channeltoken=CHANNELTOKEN

  • 请求方式

    POST

  • 输入参数

    序号 参数名 类型 释义 是否必须
    0 sid String 服务Id 是

  • 输出

    {   
    "code": "200" //200是作为服务成功返回代码,失败则为404
    "msg": "错误消息"
}

3.3.4 注册新的”服务商”

  • 服务地址

    http://<ip>:<port>/channel/sprovider/reg?channeltoken=CHANNELTOKEN

  • 请求方式

    POST

  • 输入参数

    序号 参数名 类型 释义 是否必须
    0 email String 服务商email.不能重复 是
    1 pwd String 服务商密码 是
    2 name String 服务商的名称 是

  • 输出

    {
    "did": “服务商id”,
    "name": "服务商名称",
    "email": "服务商注册时的邮件",
    "channelid": "渠道商id",   
    "code": "200" //200是作为服务成功返回代码,失败则为404
    "msg": "错误消息"
}
